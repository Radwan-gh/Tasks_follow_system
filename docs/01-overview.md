# 1. نظرة عامة على النظام

## الفكرة العامة

النظام هو **لوحة مهام تعاونية على نمط Kanban (شبيه بـ Trello)**. المستخدم يُنشئ
**لوحات (Boards)**، وكل لوحة تحتوي على **قوائم (Lists)** تمثّل أعمدة (مثل: "قيد
الانتظار"، "جارٍ العمل"، "مكتمل")، وكل قائمة تحتوي على **بطاقات (Cards)** تمثّل المهام.
يمكن سحب البطاقات وإفلاتها لإعادة ترتيبها داخل القائمة أو نقلها بين القوائم، وكذلك
إعادة ترتيب القوائم نفسها.

## المكوّنات الأساسية (Architecture)

المشروع عبارة عن **Monorepo** يُدار بواسطة Turborepo + pnpm، ويتكوّن من:

### التطبيقات (`apps/`)

- **`apps/api`** — الخادم الخلفي (Backend) مبني بإطار **NestJS** فوق **Node.js/TypeScript**،
  ويستخدم قاعدة بيانات **PostgreSQL** عبر **Prisma ORM**. هو المصدر الوحيد لتطبيق
  قواعد العمل (كل التحقّق من الصلاحيات والترتيب يحدث هنا).
- **`apps/web`** — واجهة المستخدم (Frontend) مبنية بـ **React + Vite + Tailwind**،
  وتستخدم مكتبة **dnd-kit** للسحب والإفلات و **TanStack Query** لإدارة حالة الخادم.

### الحزم المشتركة (`packages/`)

- **`packages/types`** — **المصدر الوحيد للحقيقة (Single Source of Truth)** لعقود الـ API.
  يحتوي على مخطّطات **zod** التي تصف نماذج المجال وأجسام الطلبات، ويستوردها كلٌّ من
  الخادم والواجهة كي لا تتكرّر التعريفات ولا تتباعد.
- **`packages/ordering`** — خوارزمية **الفهرسة الكسرية (Fractional Indexing)** المسؤولة
  عن حساب مفاتيح الترتيب. هذه هي أهم قطعة منطقية للصحّة في النظام (انظر
  [`06-ordering.md`](./06-ordering.md)). لا تعتمد على أي مكتبة خارجية وقت التشغيل ومُختبَرة
  بمعزل عن بقية النظام.
- **`packages/config`** — إعدادات ESLint و TypeScript المشتركة.

## المصطلحات (Glossary)

| المصطلح | الشرح |
|---|---|
| **Board (لوحة)** | الحاوية الكبرى لمشروع أو فريق؛ تملك أعضاء وقوائم |
| **List (قائمة)** | عمود داخل اللوحة يجمع بطاقات في مرحلة معيّنة |
| **Card (بطاقة)** | مهمة واحدة؛ لها عنوان ووصف وتاريخ استحقاق اختياري |
| **BoardMember (عضو لوحة)** | ربط بين مستخدم ولوحة مع دور (OWNER أو MEMBER) |
| **Position (موضع)** | سلسلة نصية قابلة للترتيب المعجمي تحدّد ترتيب القائمة/البطاقة |
| **OWNER / MEMBER** | دور المستخدم *داخل لوحة محدّدة* |
| **USER / ADMIN** | دور المستخدم *على مستوى النظام كله* |
| **Refresh Token (رمز التحديث)** | رمز طويل العمر يُستخدم لإصدار رموز دخول جديدة |

## المبادئ التصميمية الحاكمة

هذه المبادئ تتكرّر في كل أنحاء منطق العمل، ويُفترض احترامها عند أي إضافة:

1. **TypeScript من الطرف إلى الطرف**، بعقود مشتركة معرّفة مرة واحدة في `packages/types`.
2. **الخادم هو الحكم النهائي**: أي حساب حسّاس (الترتيب، الصلاحيات) يُعاد حسابه على
   الخادم داخل معاملة (Transaction)، ولا يُوثَق أبدًا بقيمة أرسلها العميل.
3. **مصدر واحد للحقيقة لكل قاعدة**: التحقّق من صلاحية اللوحة يحدث في مكان واحد فقط
   (`BoardsService.assertMembership`)، والترتيب يحدث عبر خوارزمية واحدة فقط.
4. **الحركة رخيصة**: نقل عنصر واحد يكتب صفًّا واحدًا فقط في قاعدة البيانات، دون إعادة
   ترقيم الإخوة — ما يجعل السحب آمنًا تحت التعديلات المتزامنة.

## ما هو مؤجَّل عن قصد (خارج نطاق النسخة الحالية)

النسخة الحالية هي **MVP للويب**. المزايا التالية مؤجَّلة، وكلها إضافية لاحقًا دون
الحاجة لتعديل التسلسل الحالي (لوحة ← قائمة ← بطاقة):

- التعاون اللحظي (Realtime) عبر Socket.IO.
- تطبيق الجوال (Expo/React Native).
- التسميات (Labels)، التعليقات، المرفقات، المكلّفون (Assignees)، وسجلّ النشاط.
- طبقة "منظّمة (Organization)" متعدّدة المستأجرين فوق اللوحات.
