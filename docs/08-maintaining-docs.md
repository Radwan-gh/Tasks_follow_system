# 8. تحديث التوثيق مع كل تغيير (Keeping the Docs in Sync)

هذه الوثائق تصف **منطق العمل** وليست تعليقات عابرة. قيمتها تنبع من بقائها **مطابقة
للكود**. لذلك اعتُمدت السياسة التالية.

## القاعدة الأساسية

> **أي تغيير في منطق العمل أو أي ميزة جديدة يجب أن يصاحبه تحديث لملف التوثيق المعني، في
> نفس الـ Pull Request.** التغيير غير مكتمل حتى تُحدَّث الوثيقة.

هذا يشمل: تعديل قاعدة صلاحيات، تعديل شكل طلب أو استجابة، إضافة/حذف حقل في المخطّط،
تعديل خوارزمية الترتيب، إضافة مسار (Endpoint)، أو أي قرار منطقي جديد.

## آلية التطبيق الآلي (عبر المساعد الآلي)

المشروع يستخدم Claude Code، ويقرأ المساعد ملف `CLAUDE.md` بجذر المستودع في كل جلسة.
أُضيف إلى `CLAUDE.md` قسم **"Business logic documentation (`docs/`)"** يُلزِم المساعد
بتحديث الوثيقة المناسبة عند أي تغيير في منطق العمل. بهذا يجري تحديث التوثيق **تلقائيًا
كجزء من أي تعديل يقوم به المساعد**، لا كخطوة منفصلة يسهل نسيانها.

## خريطة "أين أوثّق ماذا"

استعمل هذا الجدول لتحديد الملف الذي يجب تحديثه حسب ما لمسته من الكود:

| إذا غيّرت في الكود... | حدّث هذا الملف |
|---|---|
| `prisma/schema.prisma` أو `packages/types/src/domain.ts` (كيانات/حقول) | [`02-domain-model.md`](./02-domain-model.md) |
| `auth/*` أو منطق رموز الدخول/التحديث | [`03-authentication.md`](./03-authentication.md) |
| `boards.service.ts` (`assertMembership`) أو `admin.guard.ts` أو الأدوار | [`04-authorization.md`](./04-authorization.md) |
| `boards`/`lists`/`cards` (خدمات، متحكّمات، قواعد النقل) | [`05-boards-lists-cards.md`](./05-boards-lists-cards.md) |
| `packages/ordering/*` أو `position.util.ts` | [`06-ordering.md`](./06-ordering.md) |
| `users.service.ts`/`users.controller.ts` أو `seed.ts` | [`07-admin.md`](./07-admin.md) |
| مسار جديد، مكوّن معماري جديد، أو مصطلح جديد | [`01-overview.md`](./01-overview.md) + الملف المتخصّص |

إن أضفت **موردًا جديدًا كليًا** (مثل التعليقات أو التسميات)، أنشئ ملفًا جديدًا
`09-<الاسم>.md`، وأضِفه إلى الفهرس في [`README.md`](./README.md)، وإلى هذا الجدول.

## قائمة تحقّق قبل دمج أي تغيير (Checklist)

- [ ] هل غيّر تعديلي أي قاعدة عمل، أو شكل طلب/استجابة، أو مخطّط بيانات؟
- [ ] إن نعم، هل حدّثت الملف/الملفات المعنية في `docs/` حسب الجدول أعلاه؟
- [ ] هل ما زالت المسارات المذكورة في التوثيق مطابقة للمتحكّمات الفعلية؟
- [ ] هل أضفت أي مصطلح جديد إلى معجم [`01-overview.md`](./01-overview.md)؟
- [ ] هل الوثيقة تشرح **لماذا** اتُّخذ القرار، لا **ماذا** فقط؟

## مبادئ أسلوب الكتابة

- **العربية** لغة هذا التوثيق. أبقِ أسماء الملفات والدوال والحقول بالإنجليزية كما في
  الكود لتسهيل البحث.
- ركّز على **منطق العمل والقرارات** لا على تفاصيل التركيب/التشغيل (تلك في `README.md`
  و `CLAUDE.md`).
- أشِر إلى المصدر في الكود (اسم الملف) في بداية كل قسم، كي يسهل التحقّق والتحديث.
